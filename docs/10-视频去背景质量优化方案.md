# 视频去背景质量优化方案

## 1. 现状与问题分析

### 1.1 当前实现概览

芝绘的视频去背景功能位于 `electron/main/transparentVideoService.ts`，核心流程为：

- **抠色算法**：使用 ffmpeg 的 `colorkey` 滤镜（RGB 色彩空间）
- **参数**：
  - `similarity`：0–1，控制与背景色的匹配半径（当前由 tolerance 0–255 映射）
  - `blend`：0–1，控制过渡区半透明程度（默认 0.08）
- **输出**：WebM VP9，`yuva420p` 像素格式
- **可选模式**：`contiguous=true` 时，从视频帧四角漫水填充生成遮罩，与 colorkey 取交集，避免误删主体内部同色区域

- **单帧测试**：与整段视频一致，使用 ffmpeg colorkey（`processSingleFrameColorkey`）
- **扣色预览**：仍使用 `instantTransparencyMatting`（颜色距离 + 抗锯齿 + 羽化），与 ffmpeg 实际处理逻辑**不完全一致**，仅供参考

---

### 1.2 问题一：锯齿感明显

#### 原因分析

1. **colorkey 的二元化倾向**
   - `blend=0` 时，像素要么全透明要么不透明，边缘呈硬切，锯齿严重
   - 当前 `blend=0.08` 偏小，过渡区很窄，边缘仍偏硬

2. **yuva420p 色度下采样**
   - 4:2:0 下，色度在水平和垂直方向各下采样 2 倍
   - Alpha 通道也参与 4:2:0 下采样，边缘细节被压缩，易产生块状或锯齿感

3. **similarity 与 blend 配合不当**
   - similarity 过大：背景去除范围大，但边缘过渡区可能过宽或过窄
   - blend 过小：过渡区几乎无渐变，边缘生硬

4. **colorkey vs chromakey**
   - 当前使用 `colorkey`（RGB 空间）
   - ffmpeg 另有 `chromakey`（YUV 空间），在视频场景下有时边缘更自然，可对比测试

---

### 1.3 问题二：主体边缘残留背景色

#### 原因分析

1. **色溢（Spill）**
   - 绿幕/蓝幕等背景的反光会照到主体边缘（头发、皮肤、衣物）
   - 边缘像素实际是「主体色 + 背景色」的混合
   - colorkey 只做透明/不透明判断，**不处理色溢**，因此边缘会残留绿/蓝/黑等背景色

2. **容差与边缘的权衡**
   - tolerance 小：背景去不干净，主体周围有残留
   - tolerance 大：可能误删主体边缘，或把半透明过渡区也当成背景
   - 边缘混合区本身难以用单一容差完美划分

3. **无 Despill 步骤**
   - 专业抠图流程通常包含 Despill（去溢色）
   - ffmpeg 提供 `despill` 滤镜，当前未使用

4. **背景色检测偏差**
   - 自动检测从四角 4×4px 采样，若四角有阴影、渐变或噪点，检测到的背景色可能不准
   - 偏差会导致边缘附近颜色匹配不理想，残留增多

## 2. 解决方案

### 2.1 锯齿感优化

#### 方案 A：调整 colorkey 参数（低成本）

| 参数 | 当前 | 建议 | 说明 |
|------|------|------|------|
| blend | 0.08 | 0.12–0.2 | 增大过渡区，边缘更柔和 |
| similarity | tolerance/255 | 可略降 | 减小匹配半径，主体更完整，边缘更清晰 |

- 在 `VideoMattingPanel` 中增加 `blend` 滑块（0–0.3），供用户微调
- 默认值从 0.08 提高到 0.12–0.15

#### 方案 B：改用 chromakey（YUV 空间）

- chromakey 在 YUV 空间工作，对视频压缩格式更友好
- 色度信息与视频编码一致，边缘有时更平滑
- 需验证：chromakey 输出格式与 yuva420p 的兼容性

#### 方案 C：提高 Alpha 精度（yuva444p）

- 使用 `yuva444p` 替代 `yuva420p`，Alpha 不再下采样
- 边缘更细腻，但文件体积和编码负担增加
- 需确认 libvpx-vp9 对 yuva444p 的支持情况

#### 方案 D：后处理边缘柔化

- 在 colorkey 之后加 `boxblur` 或 `gblur` 对 alpha 通道做轻微模糊
- 可平滑锯齿，但可能略微损失锐度
- 实现：`colorkey → alphaextract → gblur → alphamerge` 等 filter_complex 组合

---

### 2.2 边缘残留优化

#### 方案 A：接入 ffmpeg despill 滤镜（推荐）

- ffmpeg 3.4+ 提供 `despill` 滤镜，用于去除绿幕/蓝幕色溢
- 典型流程：`colorkey` 抠图 → `despill` 去溢色
- 参数示例：
  - `type=green` 或 `type=blue`（按背景类型选择）
  - `mix`、`expand`、`red`、`green`、`blue`、`brightness` 等可调
- 需在抠图**之后**、输出**之前**插入 despill，且需在 RGBA 空间处理

#### 方案 B：优化背景色检测

- 增加采样帧数（如 3–5 帧）和采样点
- 对采样结果做更严格的异常值过滤（如 IQR、MAD）
- 支持用户手动指定/微调背景色（吸管或色板）

#### 方案 C：双 pass 或边缘细化

- 第一遍：colorkey 得到初步 alpha
- 第二遍：对 alpha 边缘区域（如 0.1–0.9 的像素）做局部 despill 或颜色校正
- 实现复杂度较高，可作为后续增强

#### 方案 D：提供更细的容差控制

- 将 tolerance 拆分为「背景匹配容差」和「边缘过渡容差」
- 或提供「主体保护」参数，避免过度抠除边缘

---

### 2.3 预览与输出一致性

- **单帧测试**：已统一为 ffmpeg colorkey（`processSingleFrameColorkey`），与整段视频算法一致
- **扣色预览**：仍用 `instantTransparencyMatting`（欧氏距离 + 抗锯齿 + 羽化），与 ffmpeg 行为不同，仅供参考

## 3. 实施优先级建议

| 优先级 | 方案 | 工作量 | 预期效果 |
|--------|------|--------|----------|
| P0 | 调整 blend 默认值并暴露 blend 参数 | 低 | 明显减轻锯齿 |
| P0 | 接入 despill 滤镜（绿/蓝/黑需区分 type） | 中 | 显著减少边缘残留 |
| P1 | 尝试 chromakey 替代 colorkey | 低 | 可能改善边缘质量 |
| P1 | 评估 yuva444p 替代 yuva420p | 低 | 边缘更细腻，体积增大 |
| P2 | 优化背景色自动检测 | 中 | 减少因检测不准导致的残留 |
| P2 | Alpha 边缘轻微模糊后处理 | 中 | 进一步平滑锯齿 |

## 4. 技术细节参考

### 4.1 ffmpeg colorkey 与 chromakey

- **colorkey**：RGB 空间，`colorkey=0xRRGGBB:similarity:blend`
- **chromakey**：YUV 空间，`chromakey=0xRRGGBB:similarity:blend`，可选 `yuv=1` 传入 YUV 值

### 4.2 ffmpeg despill

- 文档：<https://ffmpeg.org/ffmpeg-filters.html#despill>
- 参数：`type`（green/blue）、`mix`、`expand`、`red`、`green`、`blue`、`brightness`、`alpha`
- 需在 RGBA 格式下使用，可配合 `format=rgba` 使用

### 4.3 filter_complex 示例（colorkey + despill）

```
[0:v]colorkey=0x00ff00:0.3:0.15,format=rgba[ck];
[ck]despill=type=green:mix=0:expand=0[out]
```

（具体参数需根据实际效果调优）
