# 短漫剧剧本元素说明

本文档系统梳理**短漫剧**（短篇漫画/漫剧视频）剧本所涉及的各类元素，以便评估后续产品开发目标与功能优先级。按逻辑层次组织，尽量覆盖全面。

---

## 1. 结构层次元素

| 元素 | 说明 | 常见粒度 |
|------|------|----------|
| **项目 / 漫剧** | 一整部作品，可含多集 | 1 个 |
| **集（Episode）** | 按播出/阅读单元划分的大段落，如「第 1 集」「第 2 集」 | 若干集 |
| **场景（Scene）** | 时间、空间连续的一段戏，同一场景内一般不换景 | 若干场景/集 |
| **镜头 / 分镜（Shot）** | 单个机位或单格画面，镜头切换即分镜切 | 若干镜头/场景 |
| **镜头组（Sequence）** | 若干镜头组成的有意编排的小序列，可视为「微型蒙太奇」 | 可选 |
| **节拍（Beat）** | 剧情中的一个情绪/动作/信息单位，时长可短可长 | 微观 |

**关系示意**：项目 → 集 → 场景 → 镜头（分镜） → 节拍

---

## 2. 叙事元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **剧情概要 / 大纲** | 集或场景的摘要描述，用于整体把控 | 「小明在教室发现神秘信封」 |
| **剧本文本** | 详细台词、旁白、动作描写，可直接用于分镜与配音 | 对白 + 动作说明 |
| **故事线（Storyline）** | 单条叙事线，如主线、支线、角色线 | 主线、回忆线 |
| **情节点（Plot Point）** | 推进剧情的关键转折 | 揭秘、冲突爆发 |
| **伏笔与回收** | 前文铺垫与后文呼应 | 道具、台词、设定 |
| **冲突类型** | 人 vs 人、人 vs 环境、人 vs 自我、人 vs 命运等 | 用于 AI 生成时的约束 |
| **悬念 / 钩子** | 吸引观众继续看的设置 | 集末悬念、场景内悬念 |
| **节奏控制** | 快慢、张弛、高潮分布 | 紧张段 / 放松段交替 |

---

## 3. 角色相关元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **角色（Character）** | 剧中有名有姓、有戏份的角色 | 主角、配角、龙套 |
| **角色绑定** | 集/场景与具体角色的关联，便于素材与 TTS 选择 | 第 1 集绑定：小明、小红、老师 |
| **角色弧（Character Arc）** | 角色在整部剧中的成长与变化 | 从胆小到勇敢 |
| **角色关系** | 角色之间的社会/情感关系 | 朋友、对手、师徒 |
| **对白（Dialogue）** | 角色说出的台词 | 「你好」「我不同意」 |
| **内心独白（Inner Monologue）** | 角色内心想法，不出口 | 画外音、气泡字 |
| **旁白（Narration）** | 叙述者讲述，非角色直接发言 | 全知视角、第一人称 |
| **语气 / 情绪** | 对白的情绪标记，影响 TTS 与表演 | 愤怒、悲伤、调侃 |
| **说话对象** | 对白针对谁，影响镜头与角色站位 | A 对 B 说 |
| **沉默 / 停顿** | 无台词时段，用于情绪或节奏 | Beat、留白 |

## 4. 时间与节奏元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **时长（Duration）** | 单个素材/镜头/场景的持续时间（秒） | 3 秒、10 秒 |
| **时间顺序** | 顺叙、倒叙、插叙、闪回 | 影响镜头编排 |
| **时间跳跃** | 场景内或跨场景的时间省略 | 「三年后」「次日」 |
| **实时 vs 压缩** | 故事时间与播放时间的关系 | 1 分钟演 1 小时 |
| **节奏标记** | 快切、长镜头、慢动作等 | 影响剪辑风格 |
| **节拍时长** | 一个情绪/信息单位的建议时长 | 用于自动估算分镜时长 |

## 5. 空间与场景元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **场景类型** | 室内/室外、日/夜、具体地点 | 教室、街道、森林 |
| **场景描述** | 场景的视觉与氛围说明 | 「昏暗的教室，窗外下雨」 |
| **场景转换** | 切、淡入淡出、划像等 | 影响镜头衔接 |
| **空间关系** | 角色与物品的相对位置 | 前景、中景、背景 |
| **景深 / 层次** | 前景、中景、背景的区分与虚化 | 用于镜头与景别 |
| **环境氛围** | 光影、天气、季节、时代感 | 影响视觉与音乐 |

## 6. 镜头与视觉元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **景别** | 远景、全景、中景、近景、特写 | 决定画面信息量 |
| **机位** | 俯拍、仰拍、平视、过肩 | 影响视角与情绪 |
| **镜头运动** | 推、拉、摇、移、跟、升、降 | 对应镜头 Pan/Zoom/Tracking |
| **构图** | 三分法、对称、黄金分割、留白 | 影响画面美感 |
| **焦点 / 主体** | 画面重点，可配合虚化 | 角色、物品 |
| **光影** | 主光、辅光、轮廓光、阴影 | 氛围与情绪 |
| **色彩** | 色调、对比、饱和度、情绪色 | 冷暖、黑白、高饱和 |
| **转场效果** | 切、溶、划、擦、3D 翻转等 | 场景/镜头衔接 |

## 7. 动作与表演元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **角色动作** | 走、跑、坐、手势、表情变化 | 对应精灵图、骨骼动画 |
| **动作脚本** | 可被解析并驱动动画的指令序列 | 「走路 3 步 → 转身 → 挥手」 |
| **预设动作** | 跑步、跳跃、挥手、点头等可复用动作 | 骨骼角色一键应用 |
| **表情** | 喜、怒、哀、惊、恐等 | 表情贴图、精灵帧 |
| **视线方向** | 角色看向哪里 | 影响镜头与观众注意力 |
| **肢体语言** | 姿态、手势传达情绪 | 与对白配合 |

## 8. 音频元素

| 元素 | 说明 | 示例 |
|------|------|------|
| **对白配音（Voice-over）** | 角色对白的 TTS 或真人配音 | 与剧本对白一一对应 |
| **旁白配音** | 叙述者旁白的 TTS 或配音 | 全知视角讲述 |
| **背景音乐（BGM）** | 场景/情绪配乐 | 紧张、温馨、悬疑 |
| **音效（Sound Effect）** | 动作、环境、 UI 等声音 | 脚步声、关门、雷雨 |
| **静音** | 刻意无声音片段 | 增强张力 |
| **音量与混音** | 各轨相对音量与空间感 | 对白突出、BGM 压低 |

```
项目
 └─ 集（Episodes）

           │         └─ 关键帧（位置、缩放、旋转、骨骼等）
           └─ 转场
```

## 15. 剧本标准结构（AI 输出与可修改范围）

本节定义**结构化剧本**的标准格式，用于：① 指导 AI 按此模型输出；② 本应用的剧本编辑与语音/画面设计；③ 用户指定 AI 修改某一具体部分。  
结构由多个子结构组成，每个子结构有可寻址的 **path**（如 `episode.1.scene.2.item.1`），便于精确定位修改。  

**当前设计：暂不需要节拍（Beat）**。场景直接包含**场景内容项**（SceneContentItem），每项有类型、编剧启止时间（以场景开始为 0 点）、内容；path 为 `episode.{i}.scene.{j}.item.{k}`。节拍（Beat）为未来可选扩展。

### 15.0 场景内容类型（8 种）

| 类型 key | 中文名 | 说明 |
|----------|--------|------|
| dialogue | 对白 | 角色台词，含说话人、对象、情绪 |
| action | 动作 | 角色动作、表演说明 |
| narration | 旁白 | 叙述者讲述，非角色直接发言 |
| stage | 舞台说明 | 舞台、场景、环境描述 |
| prop | 道具说明 | 道具、物品相关说明 |
| foreground | 前景说明 | 前景元素、层次说明 |
| music | 音乐说明 | BGM、配乐说明 |
| sfx | 音效说明 | 音效、环境声说明 |

### 15.0.1 场景编辑模式

- **列表模式**：表格展示，列依次为：类型、编剧启止时间、内容（可编辑）、操作（添加到 AI 对话、删除）。
- **时间线模式**：上下 Splitter，上为时间线（按类型分层，内容条类似素材条），下为选中项编辑；时间冲突时自动创建新层；支持拖拽移动、跨层拖拽，碰撞时停止。

### 15.1 顶层结构：剧本（Script）

```json
{
  "id": "script_xxx",
  "title": "神秘信封",
  "summary": "高中生小明在教室发现神秘信封，展开一段悬疑冒险。",
  "episodes": [ /* Episode[] */ ],
  "characters": [ /* CharacterRef[] */ ],
  "tags": ["悬疑", "校园", "成长"]
}
```

| 字段 | 说明 |
|------|------|
| id | 剧本唯一标识 |
| title | 剧本标题 |
| summary | 全剧概要 |
| episodes | 集列表，有序 |
| characters | 本剧角色引用列表 |
| tags | 题材、风格等标签 |

---

### 15.2 子结构一：集（Episode）

戏剧效果标签不绑定剧集（粒度太大），仅绑定场景。

```json
{
  "id": "ep_1",
  "path": "episode.1",
  "title": "第 1 集",
  "summary": "小明在空教室发现桌上的神秘信封，拆开后看到奇怪符号。",
  "characterRefs": ["小明", "小红"],
  "scenes": [ /* Scene[] */ ]
}
```

| 字段 | 说明 |
|------|------|
| id | 集唯一标识 |
| path | 可寻址路径，如 `episode.1` |
| title | 集标题 |
| summary | 本集概要 |
| characterRefs | 本集出场角色 ID 列表 |
| scenes | 场景列表，有序 |

---

### 15.3 子结构二：场景（Scene）

**当前设计**：场景直接包含**场景内容项**（items），每项有类型、编剧启止时间、内容（见 15.3.1）。兼容旧格式 dialogues/narrations/actions，加载时迁移到 items。

```json
{
  "id": "scene_1_1",
  "path": "episode.1.scene.1",
  "title": "空教室",
  "summary": "放学后，空无一人的教室，小明发现信封。",
  "location": "教室",
  "timeOfDay": "傍晚",
  "atmosphere": "安静、略带神秘",
  "items": [ /* SceneContentItem[] */ ],
  "transition": "切",
  "dramaTags": ["悬念"]
}
```

| 字段 | 说明 |
|------|------|
| id | 场景唯一标识 |
| path | 可寻址路径，如 `episode.1.scene.1` |
| title | 场景标题 |
| summary | 场景概要 |
| location | 地点 |
| timeOfDay | 日/夜/黄昏等 |
| atmosphere | 氛围描述 |
| items | 场景内容项列表（对白、动作、旁白、舞台说明、道具说明、前景说明、音乐说明、音效说明） |
| transition | 与上一场景的转场方式 |
| dramaTags | 戏剧效果标签 |

### 15.3.1 场景内容项（SceneContentItem）

统一结构，含类型、编剧启止时间（秒，以场景开始为 0）、类型相关字段。

```json
{
  "id": "item_1_1_1",
  "path": "episode.1.scene.1.item.1",
  "type": "dialogue",
  "startTime": 0,
  "endTime": 3.5,
  "layerIndex": 0,
  "speaker": "小明",
  "target": "小红",
  "text": "你相信这世上真的存在魔法吗？",
  "emotion": "犹豫"
}
```

| 字段 | 说明 |
|------|------|
| id | 唯一标识 |
| path | 可寻址路径 |
| type | dialogue \| action \| narration \| stage \| prop \| foreground \| music \| sfx |
| startTime | 编剧起始时间（秒） |
| endTime | 编剧结束时间（秒） |
| layerIndex | 时间线模式下该类型内的分层索引（同类型同层冲突时自动递增） |
| speaker | 对白：说话人 |
| target | 对白：说话对象 |
| text | 对白/旁白：文本 |
| narratorType | 旁白：全知/第一人称主角/第一人称配角 |
| description | 动作/舞台/道具/前景/音乐/音效：描述文本 |
| characters | 动作：涉及角色 |
| emotion | 对白/旁白：情绪 |

---

### 15.4 子结构三：节拍（Beat）——当前暂不使用

节拍是最小可编辑单元，内部可含**对白**、**旁白**、**动作**等，彼此独立，便于单独修改。

**当前设计**：暂不需要节拍，场景内容项（items）直接挂在场景下（见 15.3）。节拍为**未来可选扩展**。

---

### 15.5～15.7 对白、旁白、动作（已并入 SceneContentItem）

对白、旁白、动作现为 SceneContentItem 的 type 取值，字段见 15.3.1。舞台说明、道具说明、前景说明、音乐说明、音效说明同理，均使用 description 或 text 等字段。

---

### 15.8 戏剧效果标签（dramaTags）——Tag 化

冲突、反转、悬念等以**标签**形式附着在**场景**上（剧集粒度过大，不绑定），便于筛选、统计与 AI 约束。

| 标签 key | 说明 | 可附着层级 |
|----------|------|-------------|
| conflict | 冲突 | 场景 |
| reversal | 反转 | 场景 |
| suspense | 悬念 | 场景 |
| climax | 高潮 | 场景 |
| hook | 钩子（吸引续看） | 场景 |
| foreshadow | 伏笔 | 场景 |
| payoff | 回收/呼应 | 场景 |
| tension | 紧张 | 场景 |
| relief | 舒缓 | 场景 |
| comedy | 喜剧点 | 场景 |
| tearjerker | 催泪点 | 场景 |

**冲突类型子标签**（可与 conflict 组合）：
- `conflict.person` 人 vs 人
- `conflict.environment` 人 vs 环境
- `conflict.self` 人 vs 自我
- `conflict.fate` 人 vs 命运

**示例**：
```json
"dramaTags": ["suspense", "hook", "conflict.person"]
```

### 15.9 角色引用（CharacterRef）

剧本层维护角色引用，便于绑定角色素材与 TTS。

### 15.10 用户指挥 AI 修改的指令示例

基于上述可寻址 path，用户可精确定位要修改的部分：

| 用户指令 | 对应 path / 范围 |
|----------|------------------|
| 「把第 1 集第 2 场景的对白改得更搞笑一点」 | `episode.1.scene.2` 下所有 `dialogues` |
| 「修改小明在场景 2 的台词」 | `episode.1.scene.2` 下 `dialogues` 且 `speaker=小明` |
| 「这段旁白太长了，缩短一半」 | 指定 `narration.id` 或 `path` |
| 「在场景 2 加一个反转」 | `episode.1.scene.2`，添加/调整 `dramaTags: ["reversal"]` 并补充内容 |
| 「把场景 2 的悬念强化」 | `episode.1.scene.2`，调整 `dramaTags` 与相关 `narrations`、`dialogues` |

